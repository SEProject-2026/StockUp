# StockUp - Smart Home Inventory Management

**StockUp** is a distributed mobile application designed to simplify household inventory management. By leveraging advanced image processing and real-time synchronization, it ensures that every household member has a clear, updated view of their stock and shopping needs.

## 🚀 Key Features

* **Smart Receipt Scanning & Parsing:** Automatically extract raw text from receipts using **Google Cloud Vision API**, processed by a custom-built **Logic Parser** to identify products.
* **Intelligent Shopping Recommendations:** Receive smart shopping suggestions generated by analyzing the household's historical purchase data and scanning history .
* **Dynamic Shopping Lists:** Manage a shared shopping list that updates in real-time as users add items or approve system recommendations.
* **Real-time Synchronization:** Updates to inventory or shopping lists are reflected across all household devices within 3 seconds.
* **Expiration Tracking:** Monitor product shelf life with custom "Expiring Soon" alerts and filtering options.

## 🛠 Tech Stack

* **Frontend:** React Native with TypeScript.
* **Backend:** Python FastAPI.
* **User Management:** **Supabase Auth** (External Authentication Service).
* **Database:** PostgreSQL (Cloud-based via Supabase).
* **Computer Vision:** Google Cloud Vision API and OpenCV.
* **Infrastructure:** Redis for task orchestration and real-time synchronization.

## 📂 Repository Structure

The project is organized into two primary layers to maintain a clean separation of concerns:

### **Backend (`/backend`)**
* `src/api/`: REST API endpoints and request/validation schemas.
* `src/domain/`: Core business logic and entities (User, Home, Product, Receipt, Shopping List).
* `src/infrastructure/`: Technical implementations including Google Cloud Vision integration and database repositories.
* `src/services/`: Application layer orchestrating system use-cases.

### **Frontend (`/frontend`)**
* `app/`: Navigation-based screens and user flows.
* `src/components/`: Reusable UI components.
* `src/context/`: Global state management for authentication and home context.
* `src/api/`: Typed API clients for backend communication.

## ⚙️ Setup & Configuration

To protect sensitive information, API keys and credentials are managed via environment variables.

1. Clone the Repository.
2. Environment Variables: Create a .env file in both frontend and backend directories based on the provided example.env files.

3. Local Execution:

### Backend
1. Navigate to the /backend directory.
2. Install dependencies: pip install -r requirements.txt
3. setup the local db using docker:
   docker compose up -d
4. Run the application:
   python src.main 

### Frontend
1. Navigate to the /frontend directory.
2. Install dependencies: npm install
3. Configure API URL: In src/config/api.ts, switch the base URL for local development:

   // 1. Uncomment this line for local dev:
   export const API_BASE_URL = `http://${getDevServerHost()}:8000`;

   // 2. Comment out the production line:
   // export const API_BASE_URL = `https://stockup-api-e511.onrender.com`;

4. Start the application:
   npx expo start
